# 基于 SM4 算法的文件加解密系统

> 课程实践题目 3：基于 SM4 算法采用不同加密模式实现文件加解密

## 一、项目简介

本项目使用 SM4 分组密码算法，结合多种加密模式（CBC、CFB 等），实现对任意文件的加密与解密。
程序提供图形化用户界面（GUI），支持密钥管理、模式切换、加解密效率对比等功能。

## 二、技术栈

| 层级       | 技术                                    | 说明                                  |
| ---------- | --------------------------------------- | ------------------------------------- |
| 后端       | **Go 1.26**                             | 实现 SM4 算法及文件读写、加解密逻辑   |
| 桌面框架   | **Wails v2**                            | Go + Web 前端，打包为原生桌面应用     |

## 六、SM4 算法与加密模式

### 6.1 SM4 简介
- 国家密码管理局 2012 年发布的分组密码标准（GM/T 0002-2012）
- 分组长度：**128 bit（16 字节）**
- 密钥长度：**128 bit（16 字节）**
- 32 轮非线性迭代结构，加解密对称（解密为加密轮密钥逆序）
- 核心：S 盒非线性变换 + 线性变换 L + 轮函数 F

### 6.2 选用模式 1：CBC（密码分组链接）
- **特点**：每个明文块先与前一密文块 XOR 再加密；首块使用 IV
- **优点**：相同明文块产生不同密文，隐藏明文统计特性
- **缺点**：加密无法并行；密文 1 bit 错误会影响 2 个明文块
- **填充**：PKCS#7
- **公式**：
  - 加密：`Cᵢ = E_K(Pᵢ ⊕ Cᵢ₋₁)`，`C₀ = IV`
  - 解密：`Pᵢ = D_K(Cᵢ) ⊕ Cᵢ₋₁`

### 6.3 选用模式 2：CFB（密码反馈）
- **特点**：将分组密码转化为流密码，无需填充
- **优点**：明文长度任意；加密只用 SM4 加密函数
- **缺点**：加密无法并行；自同步
- **公式**：
  - 加密：`Cᵢ = Pᵢ ⊕ E_K(Cᵢ₋₁)`，`C₀ = IV`
  - 解密：`Pᵢ = Cᵢ ⊕ E_K(Cᵢ₋₁)`

### 6.4 效率对比指标
- 加密吞吐率（MB/s）
- 解密吞吐率（MB/s）
- 不同文件大小下的耗时（1 KB / 1 MB / 100 MB / 1 GB）
- CPU 占用、内存峰值
